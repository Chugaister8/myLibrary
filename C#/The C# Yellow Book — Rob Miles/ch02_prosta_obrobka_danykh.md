# 2. Проста обробка даних

У цьому розділі ми створимо справді корисну програму (особливо якщо ви займаєтеся продажем металопластикових вікон). Ми почнемо зі створення дуже простого рішення та вивчення операторів C#, що виконують базову обробку даних. Потім ми використаємо додаткові можливості мови C# для підвищення якості рішення, що розробляється.

## 2.1 Перша програма на C#

Перша програма, яку ми розглянемо, зчитуватиме ширину і висоту вікна, а потім виводитиме кількість дерева і скла, необхідних для виготовлення вікна, що відповідатиме отвору такого розміру. Це задача, яку ми поставили перед собою у розділі 1.2.2.

### 2.1.1 Приклад програми

Мабуть, найкращий спосіб почати вивчення C# — одразу зануритись у нашу першу програму. Ось вона:

```csharp
using System;
class GlazerCalc
{
    static void Main()
    {
        double width, height, woodLength, glassArea;
        string widthString, heightString;
        widthString = Console.ReadLine();
        width = double.Parse(widthString);
        heightString = Console.ReadLine();
        height = double.Parse(heightString);
        woodLength = 2 * ( width + height ) * 3.25 ;
        glassArea = 2 * ( width * height ) ;
        Console.WriteLine ( "The length of the wood is " +
            woodLength + " feet" ) ;
        Console.WriteLine( "The area of the glass is " +
            glassArea + " square metres" ) ;
    }
}
```

*Зразок коду 01: Програма GlazerCalc*

Це коректна програма. Якщо дати її компілятору C#, вона скомпілюється, і її можна буде запустити. Фактична робота виконується двома виділеними рядками. Загалом кажучи, те, що стоїть перед цими двома рядками, пов'язане з ініціалізацією та отриманням значень для обробки. Те, що стоїть після двох рядків, пов'язане з виведенням відповіді користувачеві.

Тепер ми можемо пройтися по кожному рядку і спробувати зрозуміти, яку роль він відіграє в нашій програмі.

---

**`using System;`**

Це інструкція для компілятора C#, яка повідомляє йому, що ми хочемо використовувати речі з namespace System. Namespace — це місце, де певні імена мають значення. Namespace є і в наших розмовах: якщо я використовую namespace «Футбол» і кажу «Ця команда справжній вогонь» — це щось хороше. Якщо ж я використовую namespace «Пожежники» — це щось менш хороше.

> Велика частина навчання програмуванню — це навчання використанню всіх додаткових можливостей системи, що підтримують ваші програми.

У випадку C# namespace System містить опис безлічі корисних речей. Одна з таких корисних речей, що постачається разом з C#, — це клас Console, який дозволяє виводити речі, що з'являтимуться на екрані перед користувачем. Щоб просто посилатися на нього як на Console, потрібно повідомити компілятору, що ми використовуємо namespace System. Це означає, що якщо я посилаюся на щось за певним іменем, компілятор шукатиме в System, чи є там щось з таким іменем. Пізніше ми будемо використовувати інші namespace.

---

**`class GlazerCalc`**

Програма на C# складається з одного або кількох класів. Клас — це контейнер, що містить дані та програмний код для виконання певного завдання. У випадку нашого калькулятора для вікон клас містить лише один метод, який обчислює довжину дерева і площу скла, але клас може містити значно більше, якщо потрібно.

> Класи є основою об'єктно-орієнтованого програмування, як ми побачимо пізніше.

Для кожного класу, який ви створюєте, потрібно вигадати ідентифікатор. Наш клас я назвав GlazerCalc, оскільки це відображає те, що він робить. Поки що не надто переймайтеся класами; просто переконайтеся, що ви обираєте осмислені імена для класів, які створюєте.

І ще одна річ: є угода, що ім'я файлу, в якому міститься певний клас, повинне збігатися з назвою самого класу; іншими словами, наведена вище програма має зберігатися у файлі GlazerCalc.cs.

---

**`static`**

Це ключове слово гарантує, що метод, який іде після нього, завжди присутній, тобто слово static у даному контексті означає «є частиною класу, що оточує, і завжди тут». Коли ми дійдемо до розгляду об'єктів, ми з'ясуємо, що це маленьке ключове слово має всілякі цікаві наслідки. Але поки що я був би вдячний, якщо б ви просто переконалися, що ставите його тут, щоб ваші програми працювали правильно.

---

**`void`**

void означає «нічого». У термінах програмування ключове слово void означає, що метод, який ми збираємося описати, нічого нам не повертає. Метод просто виконає завдання і завершиться. У деяких випадках ми пишемо методи, які повертають результат (фактично ми використовуємо такий метод пізніше в програмі).

Однак, щоб запобігти випадковому використанню значення, що повертається нашим методом Main, ми явно вказуємо, що він не повертає нічого. Це робить наші програми безпечнішими: компілятор тепер знає, що якщо хтось намагається використати значення, що повертається цим методом, — це помилка.

---

**`Main`**

Ви обираєте імена методів так, щоб вони відображали те, що вони роблять для вас. За винятком Main. Саме тут (і повинен бути один і тільки один такий метод) починається виконання вашої програми. Коли ваша програма завантажується і запускається, першим отримує управління метод Main. Якщо не включити метод Main, система буквально не знає, з чого починати.

---

**`()`**

Це пара дужок, що не містить нічого. Це може здатися безглуздим, але насправді говорить компілятору, що метод Main не має параметрів. Параметр методу надає методу щось, над чим він має працювати. Коли ви визначаєте метод, ви можете сказати C#, що він працює з одним або кількома елементами, наприклад sin(x) може працювати зі значенням кута x у вигляді числа з плаваючою комою. Ми розглянемо методи дуже детально пізніше.

---

**`{`**

Це фігурна дужка. Як і передбачає назва, фігурні дужки ходять парами: для кожної відкриваючої дужки має бути відповідна закриваюча. Фігурні дужки дозволяють програмістам об'єднувати фрагменти програми. Такий фрагмент програми часто називають блоком. Блок може містити оголошення змінних, що використовуються всередині нього, за якими слідує послідовність операторів програми, що виконуються по порядку. В даному випадку фігурні дужки охоплюють робочі частини методу Main.

Коли компілятор бачить відповідну закриваючу дужку в кінці, він знає, що досяг кінця методу і може шукати інший (якщо він є). Наслідки непарної фігурної дужки завжди є фатальними…

---

**`double`**

Тепер вам, мабуть, хочеться випити. Але не це означає double в даному контексті. Це означає «число з плаваючою комою подвійної точності» (double precision floating point number).

Нашій програмі необхідно запам'ятовувати певні значення під час роботи. Зокрема, вона зчитає значення ширини і висоти вікон, а потім обчислить і виведе значення площі скла і довжини дерева. Місця, де зберігаються значення, C# називає змінними. На початку будь-якого блоку ви можете повідомити C#, що хочете зарезервувати простір для зберігання певних значень. Кожен елемент може зберігати значення певного виду. По суті, C# може обробляти три типи даних: числа з плаваючою комою, цілі числа і текст (тобто букви, цифри і знаки пунктуації). Процес створення змінної називається оголошенням змінної.

> Змінна типу double може зберігати дуже широкий діапазон значень з дуже високою точністю.

Ви оголошуєте кілька змінних певного типу, вказуючи тип даних, після якого слідує список імен, які ви хочете дати змінним. Поки що ми використовуємо тип double. Пізніше ми будемо використовувати інші типи.

---

**`width, height, woodLength, glassArea`**

Це список. Список елементів у C# розділяється символами , (кома). В даному випадку це список імен змінних. Побачивши слово double (дивіться вище), компілятор очікує знайти ім'я принаймні однієї змінної для створення. Компілятор проходить по списку, створюючи «ящики», які можуть зберігати значення типу double, і присвоює їм відповідні імена. Починаючи з цього моменту ми можемо посилатися на наведені вище імена, і компілятор знатиме, що ми використовуємо ту чи іншу змінну.

> **Порада програміста: Знайте, звідки беруться ваші дані**
>
> Враховуючи обмежену точність, з якою люди можуть знімати мірку рулеткою, і той факт, що ми не будемо виготовляти вікна завширшки з всесвіт, число з плаваючою комою подвійної точності є надлишковим для цього застосування. Замість цього ви б запитали замовника, чи можна просто виражати розміри в міліметрах. Ми розглянемо чинники, що визначають вибір конкретних типів змінних, трохи пізніше. Всі ці рішення визначаються метаданими (даними про дані), які ви збираєте під час з'ясування деталей системи, що створюється.

---

**`;`**

Крапка з комою позначає кінець списку імен змінних, а також кінець цього оператора оголошення. Усі оператори в програмах на C# відокремлюються символом ;, це допомагає компілятору не збитися зі шляху.

Символ ; насправді дуже важливий. Він говорить компілятору, де закінчується даний оператор. Якщо компілятор не знаходить його там, де очікує побачити, він видає помилку. Можна порівняти ці символи з перфораційними отворами в кіноплівці — вони синхронізують все.

---

**`string widthString, heightString;`**

Ми створили деякі змінні, що можуть зберігати числа. Тепер ми збираємося створити деякі, що можуть містити рядки. Це тому, що коли ми зчитуємо числа від користувача, ми спочатку зчитуємо їх як рядки тексту. Потім ми перетворюємо текст в число. Змінні widthString і heightString (зверніть увагу на осмислені імена) міститимуть текстові версії чисел.

---

**`widthString =`**

Це оператор присвоєння. У цьому операторі C# ми збираємося змінити значення у змінній. Наша програма зчитає рядок тексту від користувача і помістить результат у змінну, яку ми назвали widthString. Пам'ятайте, що змінна — це просто іменований ящик певного розміру, який може зберігати один елемент даних (в даному випадку рядок тексту).

Значна частина ваших програм складатиметься з інструкцій для присвоєння нових значень змінним у міру обчислення різних результатів. C# використовує символ = для виконання присвоєнь. Перша частина цього оператора — ім'я раніше визначеної змінної. За нею слідує символ =, який я називаю «занурювачем» (gozzinta). Я так його називаю, бо значення праворуч «занурюється» (goes into) у змінну ліворуч. Вибачте.

---

**`Console.`**

Праворуч від знаку рівності — те, що буде присвоєно widthString. В даному випадку результатом буде рядок, повернутий методом ReadLine. Цей метод є частиною об'єкта Console, який відповідає за введення та виведення даних користувача. Крапка (.) відокремлює ідентифікатор об'єкта від ідентифікатора методу.

---

**`ReadLine`**

Це вказує, що метод ReadLine має бути викликаний. Це просить запущену програму побігти, виконати всі оператори цього методу, а потім повернутися. Методи — це спосіб розбити програму на ряд фрагментів, кожен з яких виконує певне завдання. Вони також означають, що вам потрібно написати певний фрагмент коду лише один раз, помістити його в метод, а потім викликати цей метод будь-коли, коли вам потрібно виконати це завдання. Система C# містить ряд вбудованих методів для виконання завдань в наших програмах. ReadLine — один з них.

Коли ця програма запускається, метод ReadLine викликається. Він чекатиме, поки користувач введе рядок тексту і натисне клавішу Enter. Все, що введено, повертається як рядок методом ReadLine. Результат виклику методу поміщається у змінну widthString.

---

**`()`**

Після виклику методу слідують параметри методу. Параметр — це те, що передається в метод для обробки. Думайте про це як про сировину для якогось процесу. У випадку ReadLine немає сировини; він збирається отримати інформацію від користувача, який введе її. Однак ми все одно маємо надати список параметрів, навіть якщо він порожній.

---

**`;`**

Крапку з комою ми вже бачили. Вона позначає кінець оператора в нашій програмі.

---

**`width =`**

Це ще одне присвоєння. Змінній width надається значення. Багато операторів у ваших програмах просто переміщатимуть дані і виконуватимуть дії з ними.

---

**`double.`**

Але решта оператора виглядає трохи страшно. Насправді це не так вже й складно. Ми просимо `double` (клас, відповідальний за зберігання чисел з плаваючою комою подвійної точності) виконати для нас невелике завдання. В даному випадку невелике завдання — «взяти рядок, що зберігається у widthString, і перетворити його на число з плаваючою комою подвійної точності». Клас double надає цю можливість, відкриваючи метод Parse.

Зауважте, що немає нічого поганого чи непристойного в тому, що щось у C# відкриває свої методи. Саме так ми отримуємо виконання певних дій. Коли ви будете проєктувати великі програми, ви виявите, що найкращий спосіб зробити це — створювати компоненти, що відкривають методи для виконання завдання. Весь набір бібліотек C# надається як серія методів, і одне з завдань, з яким вам доведеться розібратися, — де знаходяться методи і як їх використовувати. Як і в багатьох справах у житті, хитрість полягає в тому, щоб знати, до кого звернутися…

---

**`Parse`**

Метод Parse виконує завдання перетворення переданого рядка на число з плаваючою комою подвійної точності. Для цього він повинен пройти по рядку, витягти кожну цифру по черзі, а потім обчислити фактичне значення, наприклад «12» означає десяток і дві одиниці. Цей процес проходу по рядку часто називається синтаксичним розбором (parsing). Звідси й назва методу. Методу передається рядок для розбору, і він повертає знайдене число.

Зверніть увагу, що це несе значний потенціал для проблем: якщо користувач нічого не введе або введе щось на кшталт «Twenty Five», виклик методу Parse не зможе витягти число і завершиться помилкою. Про те, як він завершується помилкою і як цю помилку можна вирішити, ми поговоримо в наступних розділах.

---

**`(widthString);`**

Ми бачили, що виклик методу повинен супроводжуватися сировиною (параметрами) для цього методу. У випадку ReadLine параметрів немає, але нам все одно потрібно надати порожній список. У випадку Parse методу потрібно передати рядок для обробки. Ми робимо це, поміщаючи ім'я рядкової змінної, що містить текст (widthString), у дужки. Значення інформації в widthString (тобто текст, введений користувачем) передається в метод Parse для обробки та вилучення числа.

---

**`heightString = Console.ReadLine();`**
**`height = double.Parse(heightString);`**

Ці два оператори просто повторюють процес зчитування тексту значення висоти і наступного перетворення його в значення подвійної точності, що містить висоту вікна.

---

**`woodLength = 2*(width + height)*3.25 ;`**

Це і є суть самої програми. Це та частина, яка виконує роботу. Вона бере значення висоти і ширини і використовує їх для обчислення необхідної довжини дерева.

У попередньому випадку, між двома числами з плаваючою комою подвійної точності, + означав «виконати арифметичну суму». Тут ліворуч стоїть рядок. **`woodLength`**

Це ще один приклад контексту в дії. Раніше ми використовували woodLength як числове представлення значення — в нашій програмі це довжина необхідного дерева. Однак у контексті, де воно зараз використовується (додається до кінця рядка), воно не може функціонувати таким чином.

Дуже важливо точно розуміти, що тут відбувається. Розгляньте:

```csharp

```

Це виконає числове обчислення (2.0 + 3.0) і видасть значення з плаваючою комою подвійної точності. Потім це результуюче значення отримає запит надати рядкову версію самого себе для виведення, і буде отримано:

```

```

Але рядок коду:

```csharp

```

розцінить + як конкатенацію двох рядків. Він попросить значення 3 перетворитися на рядок (звучить дивно — але саме так і відбувається), а потім виведе:

```

```

Рядок `"2.0"` має текст значення 3.0, доданий до кінця. Ця різниця в поведінці повністю обумовлена контекстом виконуваної операції.

Можна уявити, що всі змінні в нашій програмі мають мітки з метаданими, які компілятор використовує для вирішення, що з ними робити. Змінна heightString має мітку «це рядок, використовуйте плюс — і отримаєте конкатенацію». Змінна woodLength має метадані «це число з плаваючою комою подвійної точності, використовуйте плюс — і отримаєте арифметику».

---

**`)`**

Дужка позначає кінець параметра, що формується для виклику методу WriteLine. Коли метод викликається, програма спочатку збирає готовий рядок з усіх компонентів, з'єднуючи (конкатенуючи) їх для отримання єдиного результату. Потім вона передає результуюче рядкове значення в метод, який виведе його на консоль.

---

**`;`**

Крапка з комою позначає кінець цього оператора.

---

**`}`**

Тепер — дещо справді важливе. Програма по суті завершена. Ми додали всі необхідні дії. Однак компілятору ще потрібно сказати, що ми досягли кінця програми. Перша закриваюча фігурна дужка позначає кінець блоку коду, що є тілом методу Main. Блок коду починається з { і завершується }. Побачивши це, компілятор каже собі: «Це кінець методу Main».

---

**`}`**

Друга закриваюча фігурна дужка виконує не менш важливе завдання, ніж перша. Вона позначає кінець класу GlazerCalc. У C# все існує всередині класу. Клас — це контейнер для цілої низки речей, включаючи методи. Якщо ми захочемо (а ми зробимо це пізніше), ми можемо помістити кілька методів в один клас. Але поки що нам потрібен лише один метод у класі. Тому ми використовуємо другу закриваючу фігурну дужку для позначення кінця самого класу.

## 2.2 Маніпуляції з даними

У цьому розділі ми розглянемо, як писати програми, що маніпулюють даними, як значення можна зберігати, отримувати і в цілому обробляти. Це надає нам можливість виконувати частину обробки даних у програмах.

### 2.2.1 Змінні і дані

У наведеній вище програмі для розрахунку вікон ми вирішили зберігати ширину і висоту вікон у змінних типу double. Перш ніж ми зможемо просунутися далі в нашій кар'єрі програмістів, нам потрібно розглянути, що це означає і які інші типи даних ми можемо зберігати в написаних нами програмах.

Програми оперують даними. Мова програмування повинна давати вам можливість зберігати дані, які ви обробляєте, інакше вона марна. Те, що насправді означають дані, — це те, що ви як програміст вирішуєте (дивіться вище про дані).

Змінна — це іменоване місце, де ви можете щось зберегти. Можна думати про неї як про ящик певного розміру з написаним на ньому іменем. Ви обираєте ім'я так, щоб воно відображало те, що там зберігатиметься (ми використовували осмислені імена, як woodLength у наведеній вище програмі). Вам також потрібно вибрати тип змінної (конкретний розмір і форму ящика) з діапазону типів зберігання, що надає C#. Тип змінної є частиною метаданих цієї змінної.

Програми також містять літеральні значення. Літеральне значення — це просто значення у вашій програмі, яке ви використовуєте для певних цілей. Для кожного типу змінної в мові C# є спосіб виражати літеральні значення цього типу.

### 2.2.2 Зберігання чисел

При розгляді числових значень є два різновиди даних:

- **Цілі чіткі значення**, наприклад кількість овець у полі, зубів у шестірні, яблук у кошику. Вони називаються цілими числами (integers).
- **Неточні реальні значення**, наприклад поточна температура, довжина шматка мотузки, швидкість автомобіля. Вони називаються дійсними числами (reals).

У першому випадку ми можемо зберігати значення точно; у вас завжди є точна кількість цих елементів, вони цілі (integral).

> Коли ви пишете специфікацію, вас має хвилювати точність, з якою зберігатимуться значення. Надто висока точність може сповільнити машину — надто низька може призвести до використання неправильних значень.

У другому випадку ми ніколи не можемо зберегти те, що розглядаємо, точно. Навіть якщо виміряти шматок мотузки до 100 знаків після коми, все одно не вийде отримати його точну довжину — значення завжди можна отримати точніше. Це — дійсні числа. Комп'ютер є цифровим, тобто він повністю оперує бітовими шаблонами, які можна розглядати як числа. Оскільки ми знаємо, що він працює у термінах одиниць і нулів, у нього виникають проблеми зі зберіганням дійсних значень. Для обробки дійсних значень комп'ютер фактично зберігає їх з обмеженою точністю, яку ми сподіваємося, що вона є достатньою (і зазвичай так і є).

Це означає, що коли ми хочемо щось зберегти, ми повинні повідомити комп'ютеру, чи це ціле число, чи дійсне. Нам також потрібно врахувати діапазон можливих значень, які потрібно зберігати, щоб вибрати відповідний тип.

Ви повідомляєте C# про змінну, яку хочете створити, оголошуючи її. Оголошення також ідентифікує тип того, що ми хочемо зберегти. Думайте про це як про те, що C# створює ящик певного розміру, спеціально призначений для зберігання елементів даного типу. Ящик позначається метаданими (ось знову це слово), щоб система знала, що в нього можна покласти і як цей ящик можна використовувати.

#### Зберігання цілих значень

Цілі числа — найпростіший тип значень для зберігання комп'ютером. Кожне значення відображатиметься на певний бітовий шаблон. Єдиним питанням є діапазон. Чим більше значення, тим більша кількість бітів потрібна для його представлення.

C# надає різноманіття цілочисельних типів залежно від діапазону значень, який ви хочете зберігати:

> Зверніть увагу, що ми можемо зайти на одне більше у від'ємний бік, ніж у позитивний. Це тому, що числа зберігаються з використанням представлення «доповнення до двох» (2's complement).

| Тип | Розмір | Діапазон |
|-----|--------|----------|
| `sbyte` | 8 біт | від -128 до 127 |
| `byte` | 8 біт | від 0 до 255 |
| `short` | 16 біт | від -32 768 до 32 767 |
| `ushort` | 16 біт | від 0 до 65 535 |
| `int` | 32 біти | від -2 147 483 648 до 2 147 483 647 |
| `uint` | 32 біти | від 0 до 4 294 967 295 |
| `long` | 64 біти | від -9 223 372 036 854 775 808 до 9 223 372 036 854 775 807 |
| `ulong` | 64 біти | від 0 до 18 446 744 073 709 551 615 |
| `char` | 16 біт | від 0 до 65 535 |

Стандартний цілочисельний тип int може зберігати в C# надзвичайно великі числа — в діапазоні від -2 147 483 648 до 2 147 483 647. Якщо вам потрібно зберігати ще більші цілі числа (хоча я не уявляю, навіщо це може знадобитися), є тип long.

Приклад цілочисельної змінної — щось, що відстежує кількість овець у полі:

```csharp
int numberOfSheep = 0;
```

Це створює змінну, здатну відстежувати понад два мільярди овець! Вона також встановлює початкове значення 0. Використання int (який може зберігати від'ємні значення) дозволяє програмі маніпулювати «від'ємними вівцями», що, мабуть, не має сенсу (хіба що ви управляєте банком овець і даєте їх позичати). Пам'ятайте, що сама мова не усвідомлює жодних таких міркувань. Якщо ви хочете переконатися, що у нас ніколи не буде більше 1000 овець і кількість овець ніколи не стане від'ємною, ви повинні додати цю поведінку самостійно.

Якщо редагувати вихідний код програми у Visual Studio (або іншому редакторі з підсвічуванням синтаксису), ви виявите, що імена типів, вбудованих у мову C# (таких як int і float), відображаються синім, як показано вище.

> **Порада програміста: Перевіряйте власні обчислення**
>
> Ще одна річ, яку вам слід мати на увазі: програма не завжди виявить перевищення діапазону змінної. Якщо я помещу значення 255 у змінну типу byte — це нормально, тому що 255 — найбільше можливе значення цього типу. Однак якщо я додам одиницю до значення в цій змінній, система може не виявити це як помилку. Насправді це може призвести до того, що значення «обернеться» до 0. Що може спричинити серйозні проблеми для моєї програми.

#### Літеральні цілочисельні значення

Цілочисельний літерал виражається як послідовність цифр без десяткової крапки:

```
23
```

Це ціле значення 23. Я міг би використати його в програмі таким чином:

```csharp
numberOfSheep = 23 ;
```

Якщо я використовую один з «коротших» типів, літеральне значення розглядається компілятором як значення цього типу:

```csharp
sbyte tinyVal = 127;
```

В цьому операторі 127 розглядається як літерал типу sbyte, а не як ціле число. Це означає, що якщо я зроблю щось нерозумне:

```csharp
sbyte tinyVal = 128;
```

(максимальне значення, яке може зберігати sbyte, — 127), компілятор виявить, що я зробив помилку, і програма не скомпілюється.

#### Зберігання дійсних значень

«Дійсне число» (real) — загальний термін для чисел, що не є цілими. Вони мають десяткову крапку і дробову частину. Залежно від значення десяткова крапка «плаває» по числу, звідси й назва float.

C# надає тип, що може зберігати дійсне число. Стандартне значення float має діапазон від 1.5E-45 до 3.4E48 з точністю лише 7 цифр (тобто гіршою за більшість кишенькових калькуляторів).

Якщо вам потрібна більша точність (хоча ваші програми займатимуть більше пам'яті і виконуватимуться повільніше), ви можете використовувати ящик типу double (double — скорочення від «подвійна точність»). Він займає більше пам'яті, але має діапазон від 5.0E-324 до 1.7E308 і точність 15 цифр.

Приклад змінної типу float — щось, що зберігає середню ціну морозива:

```csharp
float averageIceCreamPriceInPence;
```

Приклад змінної типу double — щось, що зберігає ширину всесвіту в дюймах:

```csharp
double univWidthInInches;
```

Нарешті, якщо вам потрібна максимальна точність, але дещо менший діапазон, ви можете використовувати тип decimal. Він використовує вдвічі більше пам'яті, ніж double, і зберігає значення з точністю 28–29 цифр. Він використовується у фінансових розрахунках, де числа не такі великі, але мають зберігатися з дуже високою точністю.

```csharp
decimal robsOverdraft;
```

#### Літеральні значення дійсних чисел

Є два способи зберігання чисел з плаваючою комою: як float або як double. Що стосується самих літеральних значень у програмі, компілятор хоче знати, чи ви записуєте значення з плаваючою комою (ящик меншого розміру) або значення подвійної точності (ящик більшого розміру).

Літерал типу float виражається як дійсне число з f після нього:

```
2.5f
```

Літерал типу double виражається як дійсне число без f:

```
3.5
```

Для вираження значень double і float можна також використовувати показники степеня:

```
9.4605284E15
```

Це літеральне значення подвійної точності, що фактично є кількістю метрів у світловому році. Якщо поставити f в кінці, воно стає літеральним значенням типу float.

На відміну від цілих чисел, з дійсними числами компілятор досить прискіпливий щодо того, як їх можна і не можна поєднувати. Це тому, що при переміщенні значення зі змінної подвійної точності у звичайну змінну з плаваючою комою частина точності втрачається. Це означає, що вам доводиться вживати спеціальних заходів, щоб чітко дати зрозуміти, що саме ви хочете це зробити і що вас влаштовують наслідки. Цей процес називається приведенням типів (casting), і ми розглянемо його детально трохи пізніше.

> **Порада програміста: Прості типи змінних, мабуть, найкращі**
>
> Ви виявите, що я, як і більшість програмістів, схильні використовувати лише цілі числа (int) і числа з плаваючою комою (float). Це може здатися марнотратним (малоймовірно, що мені коли-небудь знадобиться відстежувати два мільярди овець), але це робить програми зрозумілішими.

### 2.2.3 Зберігання тексту

Іноді інформація, яку ми хочемо зберігати, є текстом. Це може бути у вигляді одного символу; в інших випадках це буде рядок. C# надає змінні для роботи з обома цими видами інформації:

#### Змінні типу char

`char` — це тип змінної, що може зберігати один символ. Символ — це те, що ви отримуєте, натискаючи клавішу на клавіатурі або відображаючи один символ на екрані. C# використовує набір символів UNICODE, що може обробляти понад 65 000 різних символів, включаючи широкий діапазон іноземних символів.

Приклад символьної змінної — щось, що зберігає клавішу команди, щойно натиснуту користувачем:

```csharp
char commandKey;
```

#### Літеральні значення типу char

Символ виражається, беручи його в одинарні лапки:

```
'A'
```

Це означає «символ A». Це те, що ваша програма отримала б, якщо попросила зчитати символ з клавіатури, а користувач натиснув Shift і A. Якщо ви редагуєте програму в редакторі з підсвічуванням синтаксису, символьний літерал відображається червоним.

#### Керівні послідовності символів

Це підводить нас до питання: «Як виразити символ ' (одинарні лапки)?» Це досягається за допомогою керівної послідовності (escape sequence). Це послідовність символів, що починається зі спеціального символу «виходу» (escape). «Вихід» у цьому контексті означає «вийти за рамки звичайних нудних угод про просте відповідність значенню і зробити щось особливе». Символ виходу — \\ (зворотна коса риска). Можливі керівні послідовності:

| Символ | Назва |
|--------|-------|
| `\'` | Одинарні лапки |
| `\"` | Подвійні лапки |
| `\\` | Зворотна коса риска |
| `\0` | Нуль |
| `\a` | Оповіщення (Alert) |
| `\b` | Повернення на крок (Backspace) |
| `\f` | Прогін сторінки (Form feed) |
| `\n` | Новий рядок (New line) |
| `\r` | Повернення каретки (Carriage return) |
| `\t` | Горизонтальна табуляція |
| `\v` | Вертикальна табуляція |

Дія цих керівних послідовностей залежить від пристрою, на який вони відправляються. Деякі системи видаватимуть звуковий сигнал при відправленні символу Alert. Деякі очищатимуть екран при відправленні символу Form feed.

Їх можна використовувати так:

```csharp
char beep = '\a' ;
```

Зверніть увагу, що літера `a` має бути в нижньому регістрі.

#### Кодові значення символів

Ми вже встановили, що комп'ютер насправді маніпулює числами, а не буквами. C# використовує стандарт Unicode для відображення символів на числа, що їх представляють. За бажанням ви можете виразити символьний літерал як значення з набору символів Unicode. Хороша новина в тому, що це надає доступ до величезного діапазону символів (за умови, що ви знаєте їх коди). Погана новина в тому, що це значення потрібно виражати в шістнадцятковій системі (hexadecimal), що трохи складніше, ніж десяткова. Найкраща новина — ймовірно, вам рідко або взагалі не доводиться цим займатися.

Як приклад: код Unicode для великої літери A — 65. У шістнадцятковій системі (основа 16) це чотири шістнадцятки і одна одиниця (тобто 41). Тому в програмі це можна записати як:

```csharp
char capitalA = '\x0041' ;
```

Зверніть увагу, що перед двома шістнадцятковими цифрами потрібно ставити ведучі нулі.

#### Рядкові змінні

Тип ящика, що може зберігати рядок тексту. У C# рядок може бути дуже коротким, наприклад `"Rob"`, або дуже довгим, наприклад `"War and Peace"` (це книга, а не три слова). Рядкова змінна може зберігати рядок тексту. Однак, оскільки є спеціальні символи, що означають «перейти на новий рядок» (дивіться вище), цілком можливо, що один рядок містить велику кількість рядків тексту.

Приклад рядкової змінної — щось, що зберігає рядок, щойно введений користувачем:

```csharp
string commandLine;
```

#### Літеральні рядкові значення

Рядковий літерал виражається в подвійних лапках:

```
"this is a string"
```

Рядок може містити керівні послідовності:

```
"\x0041BCDE\a"
```

Якщо виведемо цей рядок, отримаємо:

```
ABCDE
```

— і спробу видати звуковий сигнал.

Якщо я просто виражаю текст без керівних послідовностей або чогось незвичного, я можу повідомити компілятору, що це дослівний рядок (verbatim string). Для цього потрібно поставити @ перед літералом:

```
@"\x0041BCDE\a"
```

Якщо виведемо цей рядок, отримаємо:

```
\x0041BCDE\a
```

Це може бути корисним при вираженні шляхів до файлів. Дослівний символ має ще один трюк: він дозволяє рядковим літералам розтягуватися на кілька рядків:

```
@"The quick
brown fox
jumps over the lazy dog"
```

Це виражає рядок, що розтягується на три рядки. Переноси рядків у рядку зберігаються при збереженні.

### 2.2.4 Зберігання стану за допомогою булевих значень

Змінна типу `bool` (скорочення від boolean) — це тип ящика, що може зберігати стан «істинно» або «хибно». Іноді це все, що вам потрібно. Якщо ви зберігаєте, чи сплачена підписка, немає сенсу витрачати місце на тип, що може зберігати велику кількість можливих значень. Натомість вам потрібно лише зберігати стани true або false. Це єдині два значення, які допускає тип bool.

Приклад булевої змінної — та, що зберігає стан мережевого з'єднання:

```csharp
bool networkOK;
```

#### Літеральні булеві значення

Вони легко виражаються як true або false:

```csharp
networkOK = true ;
```

> **Порада програміста: Думайте про тип ваших змінних**
>
> Вибір правильного типу для змінної — це певне мистецтво. Я схильний використовувати зберігання з плаваючою комою лише як крайній захід. Вони вносять відсутність точності, яка мені не дуже подобається. Я вважаю, що з невеличкою кмітливістю я можу досить комфортно працювати з цілими числами. Наприклад, замість зберігання ціни товару як 1.5 фунта (що вимагає використання float) я зберігаю ціну як 150 пенсів.
>
> Також при обдумуванні того, як зберігати дані, важливо пам'ятати, звідки вони беруться. У наведеному вище прикладі я використав тип double для зберігання ширини і висоти вікна. Це справді нерозумно. Це не враховує ні необхідної точності, ні того, наскільки точно можна виміряти вікно. Я підозрюю, що продавець вікон не може вимірювати з точністю, більшою за 1 мм, і тому має сенс зберігати дані лише з такою точністю.
>
> Це ілюструє ще одне міркування, пов'язане з метаданими: коли вам потрібно зберігати числа з плаваючою комою, з'ясуйте, як вони виробляються, перш ніж вирішувати, як вони будуть зберігатися. Наприклад, вам може здатися, що для роботи зі швидкістю автомобіля потрібно зберігати значення з плаваючою комою. Однак, коли ви дізнаєтесь, що датчик швидкості дає відповіді з точністю до 1 милі на годину, завдання значно спрощується.

### 2.2.5 Ідентифікатори

У C# ідентифікатор — це ім'я, яке програміст обирає для чогось у програмі. Ім'я змінної більш правильно називається ідентифікатором. Ми побачимо інші місця, де ми створюємо ідентифікатори. C# має певні правила щодо того, що являє собою коректний ідентифікатор:

- Усі імена ідентифікаторів повинні починатися з літери.
- Після літери можуть стояти літери, цифри або символ підкреслення «_».

Великі і малі літери різні, тобто Fred і fred — різні ідентифікатори.

Ось кілька прикладів оголошень, одне з яких є недійсним (спробуйте вгадати яке і чому):

```csharp
int fred ;
float jim ;
char 29yesitsme ;
```

Одне із золотих правил програмування, поряд з «завжди використовуйте клавіатуру клавішами вгору»:

> **Завжди давайте змінним осмислені імена.**

Угода в C# для типів змінних, які ми зараз створюємо, — це змішувати великі і малі літери так, щоб кожне слово в ідентифікаторі починалося з великої літери:

```csharp
float averageIceCreamPriceInPence;
```

Це іноді називається camel case (верблюжий регістр), мабуть, через «горби» в ідентифікаторі, що утворюються великими літерами.

> **Порада програміста: Думайте про імена ваших змінних**
>
> Вибір імен змінних — ще одна навичка, над якою слід працювати. Вони повинні бути досить довгими для виразності, але не настільки довгими, щоб рядки програми стали занадто складними. Можливо, ім'я averageIceCreamPriceInPence є трохи надмірним у цьому відношенні. Але я міг би з цим жити. Пам'ятайте, що ви завжди можете розумно використовувати форматування, щоб програма виглядала нормально:
>
> ```csharp
> averageIceCreamPriceInPence =
>     computedTotalPriceInPence / numberOfIceCreams;
> ```

### 2.2.6 Присвоєння значень змінним

Отримавши змінну, нам потрібно знати, як помістити в неї щось і отримати значення з неї. C# робить це за допомогою оператора присвоєння. В операторі присвоєння є дві частини: те, що ви хочете присвоїти, і місце, куди ви хочете це помістити. Наприклад:

```csharp
class Assignment
{
    static void Main ()
    {
        int first, second, third ;
        first = 1 ;
        second = 2 ;
        third = second + first ;
    }
}
```

*Зразок коду 02: Простий приклад присвоєння*

Перша частина програми вже має бути досить зрозумілою. Усередині функції Main ми оголосили три змінні: first, second і third. Кожна з них є цілочисельною.

Останні три оператори — ті, що фактично виконують роботу. Це оператори присвоєння. Присвоєння надає значення вказаній змінній, яка повинна бути відповідного типу (зверніть увагу, що ви повинні бути розсудливими, оскільки компілятор, як ми вже знаємо, не знає і не піклується про те, що ви робите). Значення, що присвоюється, є виразом. Знак рівності посередині там головним чином для того, щоб ввести нас в оману; він не означає рівності в числовому сенсі — я люблю думати про нього як про «занурювач» (дивіться вище). «Занурювачі» беруть результат праворуч від знаку присвоєння і кидають його в ящик ліворуч, що означає, що:

```csharp
2 = second + 1;
```

— є програмістською провиною, яка спричинить усілякі неприємні помилки.

#### Вирази

Вираз — це щось, що можна обчислити для отримання результату. Потім ми можемо використовувати результат так, як нам завгодно в нашій програмі. Вирази можуть бути такими простими, як одне значення, і такими складними, як велике обчислення. Вони складаються з двох речей: операторів і операндів.

#### Операнди

Операнди — це речі, над якими оператори працюють. Зазвичай це літеральні значення або ідентифікатори змінних. У наведеній вище програмі first, second, third — ідентифікатори, а 2 — літеральне значення. Літеральне значення — це щось, що буквально міститься в коді. Літеральне значення має тип, пов'язаний з ним компілятором.

#### Оператори

Оператори — це те, що виконує роботу: вони вказують операцію, що виконується над операндами. Більшість операторів працюють з двома операндами, по одному з кожного боку. У наведеній вище програмі + — єдиний оператор.

Ось кілька прикладів виразів:

```
2 + 3 * 4
-1 + 3
(2 + 3) * 4
```

Ці вирази обчислюються (оцінюються) C# зліва направо, так само як і ви самі. І знову, як і в традиційній математиці, спочатку у виразі виконуються всі множення і ділення, а потім — додавання і віднімання. C# робить це, призначаючи кожному оператору пріоритет. Обчислюючи вираз, C# шукає в ньому всі оператори з найвищим пріоритетом і виконує їх першими. Потім шукає наступні і так далі до отримання кінцевого результату. Зверніть увагу, що перший вираз вище поверне 14, а не 20.

Якщо ви хочете вимусити порядок обчислення речей, ви можете взяти в дужки те, що потрібно обчислити першим, як у третьому прикладі. Можна вкладати дужки в дужки, якщо переконатися, що відкриваючих дужок стільки ж, скільки і закриваючих. Будучи простою душею, я схильний робити все дуже чітким, беручи все у дужки.

Мабуть, не варто надто переживати через це «обчислення виразів», як його називають пишномовні люди; загалом речі, як правило, обчислюються так, як ви й очікуєте.

Для повноти нижче наведено список усіх операторів, того, що вони роблять, і їх пріоритет (я перераховую оператори від найвищого пріоритету до нижчого):

- `-` — унарний мінус, мінус, що зустрічається C# у від'ємних числах, наприклад -1. Унарний означає «застосовується лише до одного елемента».
- `*` — множення; зверніть увагу на використання `*` замість більш математично точного, але заплутаного `×`.
- `/` — ділення; через складність зображення одного числа над іншим на екрані використовуємо цей символ.
- `+` — додавання.
- `-` — віднімання. Зверніть увагу, що ми використовуємо той самий символ, що і для унарного мінуса.

Це не повний список усіх доступних операторів, але для початку цього достатньо. Оскільки ці оператори працюють з числами, їх часто називають числовими операторами. Але, звичайно, слід пам'ятати, що деякі з них (наприклад, +) можуть використовуватися між іншими типами даних. Також можна використовувати оператори таким чином, що значення переміщаються з одного типу в інший. Це може спричиняти проблеми, як ми зараз і побачимо.

### 2.2.7 Зміна типу даних

Кожного разу, коли я переміщую значення з одного типу в інший, компілятор C# дуже зацікавлений у тому, що я роблю. Він хвилюється, чи не призведе операція, яку я збираюся виконати, до втрати даних. Він розглядає кожну операцію з точки зору «розширення та звуження» (widening and narrowing) значень.

#### Розширення та звуження

Загальний принцип C# такий: якщо ви «звужуєте» значення, він завжди попросить вас явно вказати, що саме цього ви і хочете. Якщо ви розширюєте — проблем немає.

Щоб зрозуміти значення цих термінів, можна уявити валізи. Якщо я збираюся у поїздку, я беру валізу. Якщо я вирішив перейти на меншу валізу, мені доведеться вийняти все з великої і покласти у меншу. Але в ній може не вистачити місця, і мені доведеться залишити одну зі своїх сорочок. Це «звуження».

Однак якщо я переходжу на більшу валізу — проблем немає. Більша валіза вмістить все з меншої і матиме місце для більшого.

З точки зору C# «розмір» типу — це діапазон значень (найбільше і найменше) і точність (кількість десяткових знаків). Це означає, що якщо я пишу:

```csharp
int i = 1 ;
float x = i;
```

Це працює нормально, тому що тип з плаваючою комою може зберігати всі значення, що підтримуються цілочисельним типом. Однак:

```csharp
float x = 1;
int i = x ;
```

— спричинить скаргу компілятора (навіть якщо в даний момент змінна x містить лише цілочисельне значення). Компілятор стурбований тим, що ви можете відкидати інформацію таким присвоєнням, і розглядає це як помилку.

Зверніть увагу, що це стосується і значень з плаваючою комою між собою, наприклад:

```csharp
double d = 1.5;
float f = d ;
```

— теж спричинить помилку, оскільки компілятор знає, що double є «ширшим», ніж float.

#### Приведення типів

Ми можемо змусити C# розглядати значення як значення певного типу за допомогою приведення типів (casting). Приведення — це додаткова інструкція компілятору, що змушує його розглядати значення певним чином. Значення приводиться шляхом постановки потрібного типу в дужки перед ним. Наприклад:

```csharp
double d = 1.5;
float f = (float) d ;
```

У наведеному вище коді повідомлення компілятору таке: «Мене не хвилює, що це присвоєння може призвести до втрати інформації. Я, як автор програми, беру на себе відповідальність за те, що програма працює правильно». Приведення можна розглядати як спосіб компілятора «умити руки» від проблеми. Якщо програма дає збій через втрату даних — це не тому, що компілятор зробив щось нерозумне.

Як ми бачили вище, кожен тип змінних має певний діапазон можливих значень, а діапазон значень з плаваючою комою є значно більшим, ніж для цілих. Це означає, що якщо ви робите такі речі:

```csharp
int i ;
i = (int) 123456781234567890.999 ;
```

> Я не вважаю це вадою C#. Це надає великої гнучкості, ціною припущення, що ви знаєте, що робите.

— приведення приречене на невдачу. Значення, яке потрапить в i, буде недійсним. У C# немає нічого, що перевіряє такі помилки. Ви як програміст зобов'язані переконатися, що ніколи не виходите за межі діапазонів типів даних, які використовуєте, — програма цього не помітить, але користувач точно помітить!

Приведення також може втрачати інформацію іншими способами:

```csharp
int i ;
i = (int) 1.999 ;
```

Наведений вище код бере 1.999 (яке компілюється як значення типу double) і приводить до int. Цей процес відкидає дробову частину, що означає: змінна i в результаті матиме значення 1, хоча вихідне число було значно ближче до 2. Слід пам'ятати, що таке усікання відбувається кожного разу, коли ви виконуєте приведення зі значення з дробовою частиною (float, double, decimal) до значення без неї.

#### Приведення і літеральні значення

Ми бачили, що в програму можна поміщати «літеральні» значення. Це просто значення, які ви хочете використовувати у своїх обчисленнях. Наприклад, у нашій програмі для вікон нам довелося множити довжину дерева в метрах на 3.25, щоб перевести значення довжини з метрів у фути (в одному метрі приблизно 3.25 фута).

C# уважно відстежує об'єднувані ним елементи, включаючи те, як він дозволяє використовувати літеральні значення. Це зроблено для того, щоб такі оператори, як:

```csharp
int i ;
i = 3.4 / "stupid" ;
```

— ставилися з тою зневагою, якої вони заслуговують. Компілятор C# знає, що ділити значення 3.4 на рядок `"stupid"` — безглуздо. Однак розгляньте таке:

```csharp
float x ;
x = 3.4 ;
```

Цей код виглядає абсолютно коректним. Однак це не так. Причина в тому, що літеральне значення 3.4 є значенням подвійної точності як літерал, а змінна x оголошена як з плаваючою комою. Якщо я хочу помістити літеральне значення з плаваючою комою у змінну з плаваючою комою, я можу використати приведення:

```csharp
float x ;
x = (float) 3.4 ;
```

Це призводить до того, що літерал подвійної точності перетворюється на значення з плаваючою комою, і присвоєння спрацьовує.

Щоб спростити ситуацію, творці C# додали інший спосіб вираження літерального значення з плаваючою комою в програмі. Якщо поставити f після значення, воно розглядається як значення з плаваючою комою. Це означає, що:

```csharp
float x ;
x = 3.4f ;
```

— скомпілюється правильно.

### 2.2.8 Типи даних у виразах

Коли C# використовує оператор, він приймає рішення щодо типу результату, що має бути отриманий. По суті, якщо обидва операнди є цілими, він вважає, що результат має бути цілим. Якщо обидва є числами з плаваючою комою, він вважає, що результат має бути числом з плаваючою комою. Це може призводити до проблем. Розгляньте:

```
1/2
1/2.0
```

Можна подумати, що це дасть однаковий результат. Ні. Компілятор вважає, що перший вираз, що містить лише цілі числа, повинен дати цілий результат. Тому він обчислить це як цілочисельне значення 0 (дробова частина завжди відкидається). Другий вираз, оскільки він містить значення з плаваючою комою, буде оцінено як результат з плаваючою комою подвійної точності — більш точну відповідь 0.5.

Поведінка оператора залежить від контексту використання. Пізніше ми побачимо, що оператор +, який зазвичай виконує числове обчислення, може використовуватися між рядками для їх конкатенації, тобто `"Ro" + "b"` дасть результат `"Rob"`.

Якщо вам потрібен повний контроль над конкретним видом оператора, який компілятор згенерує, програма повинна містити явні приведення для встановлення правильного контексту оператора.

```csharp
using System;
class CastDemo
{
    static void Main ()
    {
        int i = 3, j = 2 ;
        float fraction ;
        fraction = (float) i / (float) j ;
        Console.WriteLine ( "fraction : " + fraction ) ;
    }
}
```

*Зразок коду 03: Демонстрація приведення*

Приведення `(float)` у наведеному вище коді говорить компілятору розглядати значення в цілочисельних змінних як значення з плаваючою комою, щоб у результаті виводилося 1.5, а не 1.

> **Порада програміста: Приведення можуть додавати ясності**
>
> Я схильний ставити приведення навіть тоді, коли вони не потрібні — це може зробити програму зрозумілішою. Це може і не впливати на результат обчислення, але поінформує читача про те, що я намагаюся зробити.

### 2.2.9 Програми і шаблони

На цьому етапі ми можемо повернутися до нашої програми для вікон і розглянути, як вона працює. Код, який безпосередньо виконує роботу, зводиться лише до кількох рядків, які зчитують дані, зберігають їх у відповідному типі місця і потім використовують збережені значення для обчислення результату, потрібного користувачеві:

```csharp
string widthString = Console.ReadLine();
double width = double.Parse(widthString);
string heightString = Console.ReadLine();
double height = double.Parse(heightString);
woodLength = 2 * ( width + height ) * 3.25 ;
glassArea = 2 * ( width * height ) ;
Console.WriteLine ( "The length of the wood is " +
    woodLength + " feet" ) ;
Console.WriteLine("The area of the glass is " +
    glassArea + " square metres" ) ;
```

Цікаво те, що це шаблон поведінки, який можна використовувати знову і знову.

Як приклад розгляньмо іншого вашого друга, який утримує аптеку. Йому потрібна програма, що обчислює загальну вартість таблеток, які він купує, і кількість флаконів, що йому потрібні. Він вводить вартість таблеток і їх кількість. Таблетки завжди продаються у флаконах ємністю до 100 таблеток.

Ви можете дуже легко модифікувати свою програму для цього завдання; єдина складна частина — з'ясувати, скільки флаконів потрібно для певної кількості таблеток. Якщо просто розділити кількість таблеток на 100, цілочисельний розподіл дасть неправильну відповідь (для будь-якої кількості таблеток менше 100 ваша програма скаже, що потрібно 0 флаконів). Один із способів вирішити це — додати 99 до кількості таблеток перед виконанням ділення, примусивши кількість необхідних флаконів «заокруглитися вгору» при будь-якій кількості таблеток більше 0. Це робить активну частину програми такою:

```csharp
int bottleCount = ((tabletCount + 99) / 100) ;
int salePrice = bottleCount * pricePerBottle ;
```

Потім ми можемо помістити решту коду навколо цього для отримання готового рішення:

> Пам'ятайте, що при діленні двох цілих чисел результат завжди округлюється вниз і дробова частина відкидається.

```csharp
string pricePerBottleString = Console.ReadLine();
int pricePerBottle = int.Parse(pricePerBottleString);
string tabletCountString = Console.ReadLine();
int tabletCount = int.Parse(tabletCountString);
int bottleCount = ((tabletCount + 99) / 100) ;
int salePrice = bottleCount * pricePerBottle ;
Console.WriteLine ( "The number of bottles is " +
    bottleCount ) ;
Console.WriteLine( "The total price is " +
    salePrice ) ;
```

Цікаво те, що програма для аптекаря — це, по суті, лише варіація програми для продавця вікон. Обидві відповідають шаблону поведінки (зчитати дані, обробити їх, вивести результат), характерному для багатьох застосувань.

Кожного разу, коли вам доведеться написати програму, яка зчитує дані, обчислює відповіді і виводить результат, ви можете використовувати цей шаблон.

Частина майстерності програміста — визначення характеру задачі з точки зору найкращого шаблону для її вирішення.

## 2.3 Написання програми

Програми, які ми створили до цих пір, були дуже простими. Вони просто зчитують дані, щось з ними роблять і виводять результат. Однак нам потрібно буде створювати програми, що виконують більш складні завдання. Вони можуть приймати рішення на основі отриманих даних. Можуть повторювати дії, доки щось не стане істинним. Можуть зчитувати велику кількість даних і потім обробляти їх різними способами.

У цьому розділі ми розглянемо, як надати нашим програмам цей додатковий рівень складності, і поговоримо про загальну справу написання програм.

### 2.3.1 Програмне забезпечення як оповідання

Деякі люди кажуть, що написання програми трохи схоже на написання оповідання. Я не зовсім упевнений, що це правда. Я виявив, що деякі комп'ютерні посібники є художніми творами, але програми — це щось інше. Я думаю, що хоча це і не оповідання як таке, хороший текст програми має деякі характеристики хорошої літератури:

- Він повинен легко читатися. Ніде недосвідчений читач не повинен бути змушений відступати назад або звертатися до знань, які, на думку автора, вже є. Усі імена в тексті повинні бути виразними і відрізнятися одне від одного.
- Він повинен мати хорошу пунктуацію і граматику. Різні компоненти повинні бути організовані чітко і послідовно.
- Він повинен добре виглядати на сторінці. Хороша програма добре відформатована. Різні блоки повинні мати відступ, а оператори — розміщуватися на сторінці в добре оформленому вигляді.
- Повинно бути зрозуміло, хто її написав і коли вона востаннє змінювалася. Якщо ви написали щось гарне, поставте своє ім'я. Якщо ви змінюєте написане — додайте інформацію про внесені зміни і причини їх внесення.

Велика частина добре написаної програми — це коментарі, що їх програміст туди поміщає. Програма без коментарів — це трохи як літак з автопілотом, але без ілюмінаторів. Є шанс, що він доставить вас у потрібне місце, але буде дуже важко сказати, куди він летить, перебуваючи всередині.

#### Блочні коментарі

Коли компілятор C# бачить послідовність `/*`, що означає початок коментаря, він каже собі:

> «Ага! Ось шматок інформації для більш великих умів, ніж мій. Я ігноруватиму все, що іде далі, до появи `*/`, що закриває коментар».

Наприклад:

```csharp
/* Ця програма обчислює кількість скла і дерева для
   продавця металопластикових вікон. */
```

Будьте щедрими з коментарями. Вони допомагають зробити вашу програму значно зрозумілішою. Ви будете дуже здивовані, як швидко ви забудете, як змусили вашу програму працювати. Ви також можете використовувати коментарі, щоб інформувати людей про конкретну версію програми, коли вона востаннє змінювалася і чому, а також про ім'я програміста, що написав її, — навіть якщо це ви.

Якщо ви використовуєте редактор з підсвічуванням синтаксису, коментарі зазвичай відображаються зеленим кольором.

#### Однорядкові коментарі

Інша форма коментаря використовує послідовність `//`. Вона позначає початок коментаря, що простягається до кінця цього конкретного рядка коду. Вона корисна для написання короткої нотатки в кінці оператора:

```csharp
position = position + 1 ; // перейти до наступного клієнта
```

Я анотував оператор, щоб надати читачеві додаткову інформацію про те, що він насправді робить.

> **Порада програміста: Не додавайте надто багато деталей**
>
> Писати коментарі — дуже розумно. Але не перестарайтеся. Пам'ятайте, що читач вашої програми може бути знайомий з мовою C# і не потребує пояснень у надмірних подробицях:
>
> ```csharp
> goatCount = goatCount + 1 ; // додати одиницю до goatCount
> ```
>
> Я вважаю, що це просто образливо для читача. Якщо ви обрали осмислені ідентифікатори, ваша програма й так виражатиме більшу частину того, що вона робить, безпосередньо через код.

### 2.3.2 Управління потоком виконання

Наша перша програма для вікон дуже проста: вона виконується прямолінійно від першого оператора до останнього, а потім зупиняється. Часто ви будете стикатися з ситуаціями, коли ваша програма повинна змінювати поведінку залежно від отриманих даних. Загалом існує три типи потоку виконання програми:

1. Прямолінійний.
2. Обраний залежно від заданої умови.
3. Повторюваний відповідно до заданої умови.

Кожна написана програма складається з наведених вище трьох елементів і практично нічого іншого! Це можна ефективно використовувати при проєктуванні загального уявлення про те, як буде працювати ваша програма. Поки що ми розглядали лише програми, що виконуються прямолінійно і зупиняються. Шлях, яким слідує програма, іноді називається «потоком виконання» (thread of execution). Коли ви викликаєте метод, потік виконання переноситься до методу до його завершення.

#### Умовне виконання — if

Наведена вище програма для обчислення дерева і скла для продавця вікон є непоганою; насправді наш замовник, мабуть, буде нею досить задоволений. Однак вона не ідеальна. Проблема не в програмі, а в користувачеві.

Якщо ввести ширину вікна -1, програма продовжить роботу і обчислить безглуздий результат. У нашій програмі немає перевірки недійсних значень ширини і висоти. Користувач може мати підстави для скарги, якщо програма не в змозі розпізнати, що він ввів безглузде значення; насправді в судах США зараз розглядається ряд справ, де програма не змогла розпізнати недійсні дані, видала нісенітницю і завдала великої шкоди.

Ми хочемо помітити справді безглузді відповіді і повідомити користувача, що він зробив щось сумнівне. У специфікації нашої програми, яку ми надаємо замовнику, ми вказали щось на кшталт (це наші метадані):

```
Програма відкидає розміри вікон поза такими діапазонами:
ширина менше 0.5 метра
ширина більше 5.0 метрів
висота менше 0.75 метра
висота більше 3.0 метрів
```

Це означає, що ми зробили все можливе; якщо програма отримує 1 замість 10 для ширини — це проблема користувача; важливо з нашої точки зору те, що наведена вище специфікація захищає нас від судового переслідування!

Щоб дозволити нам це зробити, програма повинна помічати невірні значення і відхиляти їх. Для цього ми можемо використовувати конструкцію:

```csharp
if (умова)
    оператор або блок, виконуваний, якщо умова істинна
else
    оператор або блок, виконуваний, якщо умова хибна
```

Умова визначає, що відбувається в програмі. Що ми маємо на увазі під умовою? C# має спосіб явно вказати в програмі значення «істина» і «хиба». Ми вже бачили, що тип bool використовується для зберігання цього логічного стану.

Ми можемо створювати умови, що повертають логічний результат. Вони називаються «логічними умовами». Що є логічним. Найпростіша умова — просто значення true або false, наприклад:

```csharp
if (true)
    Console.WriteLine ( "hello mum" ) ;
```

Це дійсний код, хоча й досить марний, оскільки умова завжди істинна, тому «hello mum» завжди виводиться (зверніть увагу, що ми опустили частину else — це нормально, оскільки вона не є обов'язковою).

#### Умови і реляційні оператори

Щоб умови працювали на нас, нам потрібен набір додаткових реляційних операторів, які ми можемо використовувати в логічних виразах. Реляційні оператори працюють з операндами, як і числові. Однак будь-який вираз, що їх містить, може давати лише одне з двох значень: true або false. Доступні реляційні оператори:

**`==`** — рівність. Якщо ліва і права сторони рівні, вираз має значення true. Якщо вони не рівні — false.

```csharp
4 == 5
```

— обчислиться як false. Зверніть увагу, що не дуже осмислено порівнювати змінні з плаваючою комою на предмет рівності їх значень. Через те, що вони зберігаються з обмеженою точністю, ви можете виявити, що умови не виконуються, хоча мали б. Наприклад, рівняння:

```csharp
x = 3.0 * (1.0 / 3.0) ;
```

— цілком може дати x зі значенням 0.99999999, що означало б, що:

```csharp
x == 1.0
```

— було б хибним, хоча математично тест мав би давати true.

> Якщо ви хочете порівнювати значення з плаваючою комою, відніміть їх і перевірте, чи різниця дуже мала.

**`!=`** — нерівність. Протилежність рівності. Якщо операнди не рівні, вираз має значення true; якщо вони рівні — false. Знову ж таки, цей тест не радиться використовувати з числами з плаваючою комою.

**`<`** — менше. Якщо лівий операнд менше правого, вираз має значення true. Якщо лівий більший або дорівнює правому — false. Порівнювати числа з плаваючою комою таким способом цілком можна.

**`>`** — більше. Якщо лівий операнд більше правого, результат true. Якщо лівий менший або дорівнює правому — false.

**`<=`** — менше або рівне. Якщо лівий операнд менший або дорівнює правому — true, інакше false.

**`>=`** — більше або рівне. Якщо лівий операнд більший або дорівнює правому — true, інакше false.

**`!`** — НЕ (not). Може використовуватися для інвертування певного значення або виразу. Наприклад, можна сказати `!true`, що дає false, або `!(x==y)`, що означає те ж саме, що `(x!=y)`. Оператор `!` використовується, коли потрібно інвертувати значення виразу.

#### Комбінування логічних операторів

Іноді потрібно комбінувати логічні вирази для складніших перевірок. Наприклад, щоб перевірити коректність ширини вікна, потрібно перевірити, що вона більша за мінімум і менша за максимум. C# надає додаткові оператори для комбінування логічних значень:

**`&&`** — І (and). Якщо операнди по обидва боки від `&&` є true, результат — true. Якщо один із них false — результат false. Наприклад:

```csharp
(width >= 0.5) && (width <= 5.0)
```

— це буде true, якщо ширина відповідає нашому опису. Я взяв кожну з умов у дужки, щоб було легше розібратися, що відбувається. Однак компілятор здатний самостійно зрозуміти, що `&&` має застосовуватися між результатами двох логічних виразів, тому дужки насправді не потрібні.

**`||`** — АБО (or). Якщо хоча б один з операндів по обидва боки від `||` є true, результат — true. Вираз є false лише тоді, коли обидва операнди false. Наприклад:

```csharp
width < 0.5 || width > 5.0
```

> Це основа теореми Де Моргана.

— це буде true, якщо ширина є недійсною. Ми говоримо, що ширина недійсна, якщо вона менша за мінімум або більша за максимум. Зверніть увагу, що для інвертування умови (тобто true, коли значення недійсне) нам потрібно не тільки змінити `>` на `<=` у кожному виразі, але й замінити `&&` на `||`.

Використовуючи ці оператори разом з конструкцією if, ми можемо приймати рішення і змінювати поведінку програми відповідно до отриманих даних.

> **Порада програміста: Розбивайте умови**
>
> Якщо ви виявите, що будуєте величезні і складні умови для виконання оператора, можливо, ви зможете прояснити ситуацію, використовуючи більше ніж один оператор if і відповідним чином розставляючи відступи в коді. Це також полегшить відлагодження, тому що ви зможете спостерігати за тим, як програма покроково проходить по вашим умовам, замість того щоб іти в певному напрямку на основі величезної умови, яку потім доведеться розплутувати.

#### Використання блоків для об'єднання операторів

Ми вирішили, що якщо користувач дає значення поза дозволеним діапазоном, генерується помилка, а потім значення встановлюється на відповідний максимум або мінімум. Для цього нам потрібно виконати два оператори, що обираються виходячи з певної умови: один — для виведення повідомлення, інший — для виконання присвоєння. Це можна зробити за допомогою символів `{` і `}`. Кілька операторів, об'єднаних між символами `{` і `}`, розглядаються як один оператор, тому ми робимо наступне:

```csharp
if ( width > 5.0 )
{
    Console.WriteLine ("Width too big, using maximum\n") ;
    width = 5.0 ;
}
```

Тепер два оператори є блоком, що виконується, лише якщо width більше 5.0. Таким чином можна об'єднати сотні операторів — компілятор не проти. Також можна вкладати такі блоки коду всередину інших блоків — це називається вкладенням (nesting).

Кількість символів `{` і `}` у вашій програмі повинна збігатися, інакше ви отримаєте дивні та незрозумілі помилки, коли компілятор досягне кінця файлу всередині блоку або дійде до кінця вашої програми в середині файлу!

> **Порада програміста: Використовуйте форматування для інформування читача**
>
> Я значно полегшую розуміння, роблячи відступи для кожного конкретного блоку на кілька пробілів, тобто кожного разу, коли я відкриваю блок символом `{`, я зсуваю ліве поле трохи вправо. Тоді я можу відразу побачити, на якому рівні знаходжуся в будь-який момент. Саме так і роблять справжні програмісти.

#### Метадані, «магічні числа» і const

«Магічне число» (magic number) — це значення з особливим змістом. Воно ніколи не буде змінено всередині програми; замість цього воно є константою, що використовується в ній. Коли я пишу програму для вікон, я включу деякі «магічні числа», що задають максимальні та мінімальні значення для висот і ширин. Ці числа взяті з метаданих, які я так ретельно збирав, коли писав специфікацію програми.

Зараз я міг би просто використати значення 0.5, 5.0, 0.75 і 3.0 — але вони не несуть особливого змісту і роблять програму важкою для зміни. Якщо з якоїсь причини мій максимальний розмір скла стане 4.5 метра, мені доведеться перегляди всю програму і змінювати лише відповідні значення. Мені не подобається ідея «магічних чисел» у програмах; я б хотів замінити кожне число чимось більш осмисленим.

Ми можемо зробити це, оголосивши змінну константою, тобто таку, що ніколи не може бути змінена.

```csharp
const double PI=3.141592654;
```

Це означає, що можна робити такі речі:

```csharp
circ = rad * 2 * PI ;
```

Це і осмисленіше (ми використовуємо PI, а не анонімне значення), і робить програму швидшою у написанні. Скрізь, де ви використовуєте «магічне число», слід використовувати константу такого виду:

```csharp
const double MAX_WIDTH = 5.0 ;
```

Це робить ваші програми значно легшими для читання і значно легшими для зміни.

Існує угода, що іменам константних змінних завжди дають ВЕЛИКИМИ ЛІТЕРАМИ. Це для того, щоб, читаючи програму, можна було відрізнити визначені константи.

Тому ми можемо модифікувати нашу програму для вікон таким чином:

```csharp
using System;
class GlazerCalc
{
    static void Main()
    {
        double width, height, woodLength, glassArea;
        const double MAX_WIDTH = 5.0 ;
        const double MIN_WIDTH = 0.5 ;
        const double MAX_HEIGHT = 3.0 ;
        const double MIN_HEIGHT = 0.75 ;
        string widthString, heightString;
        Console.Write ( "Give the width of the window : " );
        widthString = Console.ReadLine();
        width = double.Parse(widthString);
        if (width < MIN_WIDTH) {
            Console.WriteLine ( "Width is too small.\n\n " ) ;
            Console.WriteLine ( "Using minimum" ) ;
            width = MIN_WIDTH ;
        }
        if (width > MAX_WIDTH) {
            Console.WriteLine ( "Width is too large.\n\n" ) ;
            Console.WriteLine ( "Using maximum" ) ;
            width = MAX_WIDTH ;
        }
        Console.Write ( "Give the height of the window : " );
        heightString = Console.ReadLine();
        height = double.Parse(heightString);
        if (height < MIN_HEIGHT) {
            Console.WriteLine( "Height is too small.\n\n" ) ;
            Console.WriteLine ( "Using minimum" ) ;
            height = MIN_HEIGHT ;
        }
        if (height > MAX_HEIGHT) {
            Console.WriteLine( "Height is too large.\n\n" ) ;
            Console.WriteLine ( "Using maximum" ) ;
            height = MAX_HEIGHT ;
        }
        woodLength = 2 * ( width + height ) * 3.25 ;
        glassArea = 2 * ( width * height ) ;
        Console.WriteLine ( "The length of the wood is " +
            woodLength + " feet" ) ;
        Console.WriteLine( "The area of the glass is " +
            glassArea + " square metres" ) ;
    }
}
```

*Зразок коду 04: Повна програма GlazerCalc*

Ця програма відповідає нашим вимогам. Вона не використовуватиме значення, несумісні з нашою специфікацією. Однак я все одно не назвав би її ідеальною. Якщо продавець вводить невірну висоту, програма просто обмежує значення і потребує повторного запуску з повторним введенням висоти.

Нам дійсно хотілося б мати спосіб багаторазово отримувати значення ширини і висоти, доки не отримаємо прийнятне. C# дозволяє нам це, надаючи конструкції циклів.

### 2.3.3 Цикли

Умовні оператори дозволяють вам щось робити, якщо задана умова є істинною. Однак часто потрібно повторювати щось, поки певна умова є істинною, або задану кількість разів.

C# має три способи зробити це, залежно від того, що саме ви намагаєтеся зробити. Зверніть увагу, що ми отримуємо три методи не тому, що нам потрібні три, а тому що вони полегшують написання програми (трохи як насадка до нашої бензопили, що дозволяє легше виконати певне завдання). Більша частина майстерності програмування полягає у виборі правильного інструменту або насадки для поточного завдання (решта — у розборі, чому інструмент не зробив того, чого ви очікували!).

У нашій програмі ми хочемо многократно отримувати числа, поки вони є недійсними, тобто введення правильного числа повинне призводити до зупинки циклу. Це означає, що якщо число введено правильно з першого разу, цикл виконається лише один раз. Можна подумати, що я тут трохи схитрував; все, що я зробив, — це змінив:

> Отримувати значення, доки не побачимо коректне

на

> Отримувати значення, поки вони некоректні

Частина мистецтва програмування — це зміна способу мислення про задачу відповідно до того, як мова програмування може бути навчена її вирішувати.

#### Цикл do–while

У нашій маленькій програмі на C# ми використовуємо конструкцію do–while, яка виглядає так:

```csharp
do
    оператор або блок
while (умова) ;
```

Це дозволяє повторювати фрагмент коду, доки умова в кінці не стане хибною. Зверніть увагу, що перевірка виконується після оператора або блоку, тобто навіть якщо перевірка заздалегідь є хибною, оператор виконається один раз.

Умова в цьому контексті — точно така ж, як умова в конструкції if, що відкриває цікаву можливість писати програми на кшталт:

```csharp
using System;
class Forever
{
    public static void Main ()
    {
        do
            Console.WriteLine ( "Hello mum" ) ;
        while ( true );
    }
}
```

*Зразок коду 05: Нескінченний цикл*

Це абсолютно коректна програма на C#. Питання, як довго вона буде виконуватися, є цікавим; відповідь містить елементи психології, ф'ючерсів на електроенергію і космології, тобто вона виконуватиметься до тих пір, доки:

1. Вам не набридне.
2. Не скінчиться електрика.
3. Всесвіт не зколапсує.

Це ситуація з бензопилою — не з потужною бензопилою. Так само, як будь-якою бензопилою можна відрізати собі ногу, якщо дуже постаратися, так і будь-якою мовою програмування можна написати програму, яка ніколи не зупиняється. Це нагадує мені мої улюблені інструкції для шампуню:

1. Намочіть волосся.
2. Нанесіть шампунь і ретельно втирайте.
3. Змийте теплою водою.
4. Повторіть.

Цікаво, скільки людей досі миють голову просто зараз?

#### Цикл while

Іноді потрібно вирішити, виконувати цикл чи ні, перш ніж його запустити. Якщо подумати про те, як працює наведений вище цикл, перевірка виконується після однократного виконання коду, що повторюється. Для нашої програми це саме те, що нам потрібно: треба запитати значення, перш ніж вирішити, чи є воно коректним. Щоб бути максимально гнучким, C# дає нам іншу форму конструкції циклу, що дозволяє виконувати перевірку першою:

```csharp
while (умова)
    оператор або блок
```

Зверніть увагу, що C# намагається зменшити кількість клавіш, які потрібно натискати, прибравши слово do. (Якщо ви поставите do, компілятор з великим задоволенням видасть вам повідомлення про помилку — але ви вже, звісно, здогадалися!).

#### Цикл for

Часто потрібно повторювати щось задану кількість разів. Розглянуті нами конструкції циклів можна використовувати для цього досить легко:

```csharp
using System;
class WhileLoopInsteadOfFor
{
    public static void Main ()
    {
        int i ;
        i = 1 ;
        while ( i < 11 )
        {
            Console.WriteLine ( "Hello mum" ) ;
            i = i + 1 ;
        }
    }
}
```

*Зразок коду 06: Цикл while замість for*

> Змінна, що управляє циклом, часто називається управляючою змінною і зазвичай отримує ім'я i.

Ця безглузда програма виводить «hello mum» 10 разів. Для цього вона використовує змінну для управління циклом. Змінній надається початкове значення (1), після чого вона перевіряється кожного разу, коли ми виконуємо черговий прохід циклу. Після кожного проходу управляюча змінна збільшується. Зрештою вона досягне 11, після чого цикл завершується і наша програма зупиняється.

C# надає конструкцію для такого налаштування циклу з одного разу:

```csharp
for ( ініціалізація ; умова_завершення ; оновлення )
{
    дії, що потрібно повторити задану кількість разів
}
```

Ми могли б використати це для переписування наведеної вище програми:

```csharp
using System;
class ForLoop
{
    public static void Main ()
    {
        int i ;
        for ( i = 1 ; i < 11 ; i = i + 1 )
        {
            Console.WriteLine ( "Hello mum" ) ;
        }
    }
}
```

*Зразок коду 07: Цикл for*

Ініціалізація поміщає початкове значення в управляючу змінну. Умова завершення — умова, яка має бути істинною для продовження виконання циклу for. Оновлення — оператор, що виконується для оновлення управляючої змінної в кінці кожної ітерації циклу. Зверніть увагу, що три елементи відокремлені крапками з комою. Точна послідовність подій така:

1. Помістити початкове значення в управляючу змінну.
2. Перевірити, чи ми вже завершили цикл, і, якщо так, вийти до оператора після циклу for.
3. Виконати оператори, що повторюються.
4. Виконати оновлення.
5. Повторити з кроку 2.

Написання циклу таким чином є швидшим і простішим, ніж використання while, оскільки всі елементи циклу зібрані в одному місці, а не розкидані по програмі. Це означає, що ви з меншою ймовірністю забудете зробити щось на кшталт надання початкового значення управляючій змінній або її оновлення.

Якщо ви настільки нерозумні, щоб маніпулювати значенням управляючої змінної всередині циклу, — чекайте, що ваша програма буде робити нерозумні речі: якщо, наприклад, встановити i назад до 0 всередині циклу, він виконуватиметься вічно. І поділом заслуженим.

> **Порада програміста: Не будьте розумнішими/дурнішими, ніж треба**
>
> Деякі люди люблять показувати свою кмітливість, роблячи хитрі речі з операторами ініціалізації, умови і оновлення, які можуть робити не просто присвоєння, інкремент і перевірку. Деякі програмісти вважають себе дуже розумними, якщо можуть виконати всю роботу «всередині» частини for зверху і мати порожній оператор після неї.
>
> Я називаю таких людей «тупими людьми». Рідко є потреба в такому заплутаному коді. Коли ви пишете програми, вас повинні турбувати дві речі: «Як я можу довести, що це працює?» і «Наскільки цей код легко зрозуміти?» Складний код не допомагає зробити ні те, ні інше.

#### Вихід з циклу

Іноді потрібно вийти з циклу, перебуваючи в середині нього, тобто програма може вирішити, що немає сенсу або потреби продовжувати, і хоче вистрибнути з циклу і продовжити виконання програми від оператора після нього.

Це можна зробити за допомогою оператора break. Це команда негайно залишити цикл. Зазвичай програма приймає якесь рішення про таке «втечу». Я вважаю його найбільш корисним для надання можливості «аварійного виходу» в середині чогось. Наприклад, у наступному фрагменті програми змінна aborted, зазвичай false, стає true, коли цикл потрібно перервати, а змінна runningOK, зазвичай true, стає false, коли час завершуватися нормально.

```csharp
while (runningOK)
{
    складні дії
    ....
    if (aborted)
    {
        break ;
    }
    ....
    ще складніші дії
    ....
}
....
частина, до якої ми доходимо, якщо aborted стало true
....
```

Зверніть увагу, що ми використовуємо дві змінні як перемикачі; вони не зберігають значення як такі, а фактично використовуються для представлення станів у програмі в процесі виконання. Це стандартний прийом програмування, який ви знайдете дуже корисним.

Можна виходити з будь-якого з трьох видів циклів. У кожному випадку програма продовжує виконання від оператора, що слідує за останнім оператором циклу.

> **Порада програміста: Будьте обережні зі своїми break**
>
> Ключове слово break дещо нагадує сумнозвісний оператор goto, якого програмісти часто бояться. goto — це спеціальний оператор, що дозволяє переходити від однієї частини програми до іншої. З цієї причини goto засуджується як потенційно небезпечний і заплутаний пристрій. Оператор break дозволяє перейти з будь-якої точки циклу до оператора безпосередньо після циклу. Це означає, що якщо моя програма знаходиться в операторі, що безпосередньо слідує за циклом, вона могла потрапити туди кількома способами — по одному для кожного break у наведеному вище циклі. Це може ускладнити розуміння коду. Конструкція break менш заплутана, ніж goto, але все одно може спричиняти проблеми. В зв'язку з цим ми рекомендуємо проявляти обережність при її використанні.

#### Повернення на початок циклу

Час від часу потрібно повернутися на початок циклу і виконати все знову. Це відбувається, коли ви пройшли по операторах настільки, наскільки потрібно. C# надає ключове слово continue, яке говорить приблизно наступне:

> Будь ласка, не продовжуйте виконання по цьому проходу. Поверніться на початок циклу, виконайте всі оновлення тощо і продовжуйте, якщо потрібно.

У наступній програмі булева змінна Done_All_We_Need_This_Time встановлюється в true, коли ми пройшли по циклу настільки далеко, наскільки потрібно.

```csharp
for ( item = 1 ; item < Total_Items ; item=item+1 )
{
    .....
    обробка елемента
    ....
    if (Done_All_We_Need_This_Time) continue ;
    ....
    додаткова обробка елемента
    ....
}
```

continue призводить до повторного виконання циклу з наступним значенням item, якщо це допустимо. Можна розглядати це як перехід до кроку 2 у наведеному вище списку.

#### Складніші рішення

Тепер ми можемо подумати про використання циклу для перевірки коректної ширини або висоти. По суті, ми хочемо продовжувати запитувати у користувача значення, доки не отримаємо прийнятне; тобто якщо значення більше максимального або менше мінімального — просити ще раз.

Для цього потрібно поєднати два тести, щоб перевірити, чи є значення коректним. Наш цикл повинен продовжуватися, якщо:

```
ширина > MAX_WIDTH або ширина < MIN_WIDTH
```

Для виконання цього тесту ми використовуємо один з логічних операторів, описаних вище, щоб написати умову, яка буде true, якщо ширина є недійсною:

```csharp
if ( width < MIN_WIDTH || width > MAX_WIDTH ) ..
```

> **Порада програміста: Звикайте до інвертування умов**
>
> Одна з речей, з якими вам доведеться змиритися, — це те, що часто доводиться переглядати ситуацію. Замість того щоб говорити «Дай мені коректне число», вам доведеться казати «Отримуй числа, поки вони некоректні». Це означає, що ви часто перевірятимете на наявність того, чого не хочете, а не того, що хочете. Пам'ятайте наведені вище нотатки про інвертування умов, коли будете писати код.

#### Повна програма для вікон

Це повне рішення задачі з використанням усіх розглянутих прийомів.

```csharp
using System;
class GlazerCalc
{
    static void Main()
    {
        double width, height, woodLength, glassArea;
        const double MAX_WIDTH = 5.0 ;

### 2.3.4 Скорочений запис операторів

До цих пір ми розглядали оператори, що з'являються у виразах і працюють з двома операндами, наприклад:

```csharp
window_count = window_count + 1
```

В даному випадку оператором є `+`, що діє на змінну window_count і значення 1. Мета наведеного оператора — додати 1 до змінної window_count. Однак це досить громіздкий спосіб вираження, як з точки зору того, що потрібно набирати, так і з точки зору того, що комп'ютер фактично зробить при виконанні програми. C# дозволяє нам бути лаконічнішими, якщо хочемо:

```csharp
window_count++
```

— зробить те ж саме. Ми можемо висловитися стисліше, і компілятор може генерувати ефективніший код, тому що тепер знає, що ми додаємо одиницю до конкретної змінної. `++` називається унарним оператором, тому що він працює лише з одним операндом. Він призводить до збільшення значення цього операнда на одиницю. Є відповідний оператор `--`, що можна використовувати для зменшення (декременту) змінних.

Приклади цієї конструкції можна побачити у визначенні циклу for у наведеному вище прикладі.

Інша скорочена форма запису використовується, коли ми додаємо певне значення до змінної. Можна написати:

```csharp
house_cost = house_cost + window_cost;
```

Це цілком нормально, але знову досить громіздко. C# має додаткові оператори, що дозволяють скоротити це до:

```csharp
house_cost += window_cost;
```

Оператор `+=` поєднує додавання і присвоєння, так що значення в house_cost збільшується на window_cost. Деякі інші оператори скорочення:

| Оператор | Значення |
|----------|----------|
| `a += b` | значення в a замінюється на a + b |
| `a -= b` | значення в a замінюється на a – b |
| `a /= b` | значення в a замінюється на a / b |
| `a *= b` | значення в a замінюється на a * b |

Є й інші оператори комбінування; решту я залишаю вам знайти самостійно!

#### Оператори і значення

Одна з дійсно класних речей у C# — те, що всі оператори повертають значення, яке ви можете використовувати в іншому операторі, якщо хочете. Найчастіше ви будете ігнорувати це значення, що нормально, але іноді воно може бути дуже корисним, особливо коли ми підійдемо до прийняття рішень. Щоб показати, як це робиться, розгляньте наступне:

```csharp
i = (j=0);
```

Це абсолютно коректний (і навіть, можливо, осмислений) код на C#. Він встановлює і i, і j рівними 0. Оператор присвоєння завжди повертає значення, що присвоюється (тобто частину праворуч від «занурювача»). Це значення потім можна використовувати як значення або операнд. Якщо ви це робите, радимо брати в дужки оператор, що використовується як значення — це значно прояснює ситуацію для вас і компілятора!

Коли розглядаються оператори на кшталт `++`, виникає можлива двозначність: незрозуміло, отримуєте ви значення до чи після інкременту. C# надає спосіб отримати те чи інше значення залежно від потрібного ефекту. Визначити, чи хочете ви бачити значення до чи після суми, можна за допомогою позиції `++`:

- `i++` — «Дай мені значення до інкременту»
- `++i` — «Дай мені значення після інкременту»

Наприклад:

```csharp
int i = 2, j ;
j = ++i ;
```

— зробить j рівним 3. Інші спеціальні оператори (`+=` тощо) всі повертають значення після виконання оператора.

> **Порада програміста: Завжди прагніть до простоти**
>
> Не захоплюйтеся цим. Те, що можна створювати код на кшталт:
>
> ```csharp
> height = width = speed = count = size = 0 ;
> ```
>
> — не означає, що слід. Зараз, коли я пишу програму, моє головне міркування — наскільки легко її зрозуміти. Я не думаю, що наведений вище оператор дуже легко зрозуміти, тому, незалежно від того, наскільки він ефективніший, я все одно так не роблю.

### 2.3.5 Акуратніше виведення

Якщо ви запускали будь-яку з наведених вище програм, ви, мабуть, вже помітили, що спосіб виведення чисел залишає бажати кращого. Цілі числа, здається, виводяться нормально, але числа з плаваючою комою живуть ніби своїм власним розумом. Щоб подолати це, C# надає дещо інший спосіб виведення чисел. Він забезпечує більшу гнучкість і є простішим у використанні, якщо ви виводите велику кількість значень.

> Зверніть увагу, що спосіб виведення числа не впливає на те, як воно зберігається програмою; він лише говорить методу виведення, як його слід виводити.

#### Використання заповнювачів у рядках виведення

Заповнювач просто позначає місце, де буде виводитися значення. Розгляньмо:

```csharp
int i = 150 ;
double f = 1234.56789 ;
Console.WriteLine ( "i: {0} f: {1}", i, f ) ;
Console.WriteLine ( "i: {1} f: {0}", f, i ) ;
```

Це виведе:

```
i: 150 f: 1234.56789
i: 150 f: 1234.56789
```

Частина `{n}` рядка означає «параметр номер n, починаючи з 0». У другому операторі виведення я поміняв порядок чисел, але оскільки я також поміняв порядок параметрів, виведення однакове.

Якщо зробити щось безглузде, наприклад використати `{99}` для отримання 99-го параметра, метод WriteLine завершиться з помилкою. Однак компілятор цю помилку не виявить; програма завершиться аварійно під час виконання.

#### Налаштування точності дійсних чисел

До заповнювачів можна додавати інформацію про форматування:

```csharp
int i = 150 ;
double f = 1234.56789 ;
Console.WriteLine ( "i: {0:0} f: {1:0.00}", i, f ) ;
```

Це виведе:

```
i: 150 f: 1234.57
```

Символи `0` замінюють одну або кілька цифр. Розміщені після десяткової крапки, вони можуть використовуватися для управління кількістю десяткових знаків, що використовуються для представлення значення. Зверніть увагу, що при такому форматуванні ціле число виводиться як 12.00.

#### Вказівка кількості виведених цифр

Можна вказати конкретну кількість цифр, поставивши задану кількість нулів:

```csharp
int i = 150 ;
double f = 1234.56789 ;
Console.WriteLine ( "i: {0:0000} f: {1:00000.00}", i, f );
```

Це виведе:

```
i: 0150 f: 01234.57
```

Зверніть увагу, що при такому форматуванні виводяться ведучі нулі, що корисно при виведенні, наприклад, чеків.

#### Справді витончене форматування

Якщо потрібен дійсно тонкий рівень управління, можна використовувати символ `#`. Символ `#` у рядку формату означає «вивести сюди цифру, якщо вона є»:

```csharp
int i = 150 ;
double f = 1234.56789 ;
Console.WriteLine ( "i: {0:#,##0} f: {1:##,##0.00}", i, f );
```

Я використав символ `#`, щоб виводити тисячі з комами:

```
i: 150 f: 1,234.57
```

Зверніть увагу, що форматувальник використовує лише ті символи `#` і коми, які потрібні. У числа 150 немає розряду тисяч, тому він і кома опускаються. Зверніть також, що я включив `0` як найменшу цифру. Це потрібно для того, щоб при виведенні значення 0 справді виводилася цифра, інакше при виведенні нуля я не отримаю нічого на сторінці.

#### Виведення в стовпчики

Нарешті, до інформації про форматування виведення можна додати значення ширини. Це дуже корисно для виведення матеріалу в стовпчики:

```csharp
int i = 150 ;
double f = 1234.56789 ;
Console.WriteLine ( "i: {0,10:0} f: {1,15:0.00}", i, f ) ;
Console.WriteLine ( "i: {0,10:0} f: {1,15:0.00}", 0, 0 ) ;
```

Це виведе:

```
i:        150 f:        1234.57
i:          0 f:           0.00
```

Ціле значення виводиться у стовпчик шириною 10 символів, а double — у стовпчик шириною 15 символів. Наразі виведення вирівнюється по правому краю; якщо потрібно вирівнювання по лівому краю, ширина вказується від'ємною:

```csharp
int i = 150 ;
double f = 1234.56789 ;
Console.WriteLine ( "i: {0,-10:0} f: {1,-15:0.00}", i, f ) ;
Console.WriteLine ( "i: {0,-10:0} f: {1,-15:0.00}", 0, 0 ) ;
```

Це виведе:

```
i: 150        f: 1234.57
i: 0          f: 0.00
```

Зверніть увагу, що це вирівнювання спрацює навіть у випадку виведення рядка, а не числа, — тому для виведення стовпчиків слів можна використовувати цю техніку. Можна вказати ширину виведення для будь-якого елемента, навіть для тексту, що робить виведення в стовпчики дуже простим.
